  • Cada pergunta esteja vinculada ao seu tpico de negcio

  • Toda resposta tenha origem verificvel e auditvel

  1. Auditoria completa e rastrevel

  3. Governana e versionamento seguro

1.1 Base Cannica Conhecimento Oficial do Negcio

Nesta etapa, consolidamos todo o contedo autorizado pelo negcio em um formato nico, simples e verificvel. Cada entrada segue um padro obrigatrio:

  • topic: o contexto do assunto

  • question: como o usurio formula a dvida

  • answer: a resposta oficial, aprovada pelo negcio

Esse catlogo estruturado representa a verdade auditvel do domnio. A IA no inventa contedo: ela s pode responder a partir desta base, garantindo alinhamento com regras internas, conformidade legal (incluindo LGPD) e preciso de comunicao com o usurio final.

Privacidade tratamento LGPD e gesto de consentimento
Suporte canais oficiais para dvidas e incidentes no portal

Este domnio envolve respostas sensveis que devem refletir fielmente as polticas e regras internas da empresa sem improviso, sem especulao e sem gerao livre de contedo.

def validate_kb_json(json_path: str) -> List[Dict[str, str]]:
    """
    Funo de GOVERNANA da Base Cannica.

    Ela verifica se o JSON de conhecimento est consistente e seguro para ser usado pela IA.
    Em resumo, confirma que:
     O arquivo existe e  JSON vlido
     O contedo  uma LISTA de itens (cada item = pergunta/ resposta oficial)
     Cada item contm os campos obrigatrios:
        - topic
        - question
        - answer
     Todos estes campos so strings (compatveis com embeddings e busca semntica)

    Se houver qualquer erro:
     A execuo  interrompida com uma mensagem clara de onde est o problema

    Se estiver tudo certo:
     O arquivo  carregado e retornado para uso seguro no pipeline
    """

    # Tenta abrir o arquivo JSON e converter para objeto Python
    try:
        with open(json_path, "r", encoding="utf-8") as f:
            data = json.load(f)

    except Exception as e:
        # Falha ao abrir ou ler o JSON  interrompe com erro descritivo
        raise ValueError(f" JSON invlido ou no pode ser carregado: {e}")

    # A base deve ser uma LISTA (conjunto de registros cannicos)
    if not isinstance(data, list):
        raise ValueError(" A base deve ser uma lista de entradas JSON.")

    # Campos obrigatrios para cada item do conhecimento oficial
    required_fields = {"topic", "question", "answer"}

    # Validao linha por linha (item a item)
    for i, item in enumerate(data):

        # Cada item deve ser um objeto JSON (dicionrio)
        if not isinstance(item, dict):
            raise ValueError(f" Item {i} no  um objeto JSON.")

        # Verifica se os trs campos existem
        missing = required_fields - set(item.keys())
        if missing:
            raise ValueError(f" Item {i} est faltando os campos: {missing}")

        # Verifica se os campos so strings (requisito da IA para embeddings)
        for field in required_fields:
            if not isinstance(item[field], str):
                raise ValueError(
                    f" Item {i}: campo '{field}' deve ser string, mas recebeu {type(item[field])}"
                )

    # Se chegar at aqui  tudo est correto
    print(f" JSON vlido! Registros: {len(data)}")
    return data

def buscar_topk(query: str, top_k: int = 3):
    """
    Retorna as top_k perguntas mais similares, com score de similaridade.
    """
    assert "encoder" in globals(), "Modelo 'encoder' no carregado."
    q_vec = encoder.encode([query], convert_to_numpy=True, normalize_embeddings=True).astype("float32")

    distances, idxs = nn_index.kneighbors(q_vec, n_neighbors=top_k)
    sims = 1.0 - distances[0]  # converte distncia em similaridade

    return [
        {
            "rank": r,
            "score_cosine": float(s),
            "qid": int(df_kb.iloc[i]["qid"]),
            "topic": df_kb.iloc[i]["topic"],
            "question": df_kb.iloc[i]["question"],
            "answer": df_kb.iloc[i]["answer"]
        }
        for r, (i, s) in enumerate(zip(idxs[0], sims), start=1)
    ]

def calcular_margin(resultados: list) -> float:
    """Diferena entre Top-1 e Top-2 (quanto menor, maior ambiguidade)."""
    return 1.0 if len(resultados) < 2 else float(resultados[0]["score_cosine"] - resultados[1]["score_cosine"])

2.10 Estrutura de Fluxo do Mecanismo

Pergunta do usurio
        
        
[Codificao]  encoder(SBERT)  embedding(q)
        
        
[Busca Semntica]  kNN(cosseno) sobre KB (top_k)
        
        
[Mtricas]  Top-1 (confiana)    Margin = Top1  Top2
        
        
[Gate de Deciso - Nvel 3]
    Se Top-1  TOP1_LIMIAR e Margin  MARGIN_LIMIAR    Resposta oficial
    Se Top-1  TOP1_LIMIAR e Margin <  MARGIN_LIMIAR   Dvida (ambiguidade)
    Se Top-1 <  TOP1_LIMIAR                            Fora de Escopo (OOS)

O mecanismo recebe a pergunta do usurio e a transforma em um vetor semntico usando um modelo pr-treinado (SBERT). Em seguida, realiza uma busca por similaridade na base cannica, identificando as perguntas mais prximas e calculando dois indicadores de segurana: Top-1 (confiana) e Margin (ambiguidade). Com base nesses valores, o Gate de Deciso determina se a IA pode responder com segurana, se precisa solicitar refinamento ou se deve recusar por falta de conhecimento.

Assim, o fluxo garante que o sistema responde somente quando sabe e nunca inventa reduzindo o risco tcnico, jurdico e reputacional.

TOP1_LIMIAR = 0.60
MARGIN_LIMIAR = 0.05

def decidir_resposta(resultados: list) -> dict:
    """
    Aplica as regras de deciso usando Top-1 e Margin.
    """
    if not resultados:
        return {"status": "oos", "mensagem": "No encontrei informaes suficientes."}

    r1 = resultados[0]
    score = r1["score_cosine"]
    margin = r1["margin"] if r1["margin"] is not None else 1.0

    if score >= TOP1_LIMIAR and margin >= MARGIN_LIMIAR:
        return {
            "status": "ok",
            "resposta": r1["answer"],
            "qid": r1["qid"],
            "score": score,
            "margin": margin
        }
    elif score >= TOP1_LIMIAR and margin < MARGIN_LIMIAR:
        return {
            "status": "duvida",
            "mensagem": "Preciso que voc seja mais especfico(a). Pode reformular?",
            "tops": resultados[:2],
            "score": score,
            "margin": margin
        }
    else:
        return {
            "status": "oos",
            "mensagem": "Essa informao no est na nossa base oficial. Pode tentar outra pergunta?",
            "score": score,
            "margin": margin
        }

    decisao = decidir_resposta(resultados)

def search_questions(query: str, k: int = 5):
    """Retorna Top-K candidatos com score cosseno."""
    qv = encoder.encode([query], normalize_embeddings=True).astype("float32")
    D, I = index.search(qv, min(k*5, len(df)))
    pairs = [(int(I[0,j]), float(D[0,j])) for j in range(I.shape[1]) if I[0,j] != -1]
    pairs = pairs[:k]

    resultados = []
    for qid, score in pairs:
        row = df.iloc[qid]
        resultados.append({
            "qid": qid,
            "score": round(score, 4),
            "topic": row["topic"],
            "question": row["question"],
            "answer": row["answer"],
        })
    return resultados


def decide_from_scores(query: str, results):
    """Combina verificao lexical + semntica + limiares de confiana."""

    # 1) Hard-OOS
    if is_hard_oos(query):
        return {"decision": "oos", "message": "Assunto fora do escopo."}

    if not results:
        return {"decision": "oos", "message": "Sem candidatos relevantes."}

    # Ordenao
    cands = sorted(results, key=lambda x: x["score"], reverse=True)
    top1, top2 = cands[0], (cands[1] if len(cands)>1 else {"score":0.0})
    margin = float(top1["score"] - top2["score"])

    #  Gate por centride (domnio)
    qv = encoder.encode([query], normalize_embeddings=True).astype("float32")
    centroid_scores = (qv @ C.T).flatten()
    if float(max(centroid_scores)) < CENTROID_MIN:
        return {"decision": "oos", "message": "Pergunta no reconhecida no domnio."}

    # 3) Score absoluto baixo
    if top1["score"] < 0.50:
        return {"decision": "oos", "message": "Confiana insuficiente."}

    # 4) Query genrica  exigncia extra
    if not is_domain_like(query) and top1["score"] < 0.65:
        return {"decision": "oos", "message": "Pergunta muito genrica."}

    # 5) Ambiguidade
    if margin < 0.05:
        return {"decision": "two_options", "options": cands[:2], "margin": round(margin,4)}

    #  Confiana alta
    if top1["score"] >= TOP1_LIMIAR_OK and margin >= MARGIN_MIN:
        return {"decision": "answer", "best": top1, "margin": round(margin,4)}

    #  Confiana moderada
    if top1["score"] >= TOP1_LIMIAR_DUV:
        return {"decision": "two_options", "options": cands[:2], "margin": round(margin,4)}

    return {"decision": "oos", "message": "Fora do domnio."}
